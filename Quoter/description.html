Introduction

This sample has been updated to work with Roslyn September 2012 CTP and requires Visual Studio 2012 and .NET 4.5. Last updated: 11/20/2012 - fixed a few bugs.
 
With the Microsoft "Roslyn" CTP sometimes you need to manually construct parts of the syntax tree using the static factory methods on the Roslyn.Compilers.CSharp.Syntax class. Sometimes it is not obvious which API need to be called to construct syntax nodes that result in the desired C# program.
The Quoter tool provides a Quote method that accepts a string and returns a string. The input string is any C# program (a single compilation unit). It does not have to be syntactically correct. The output is the text of a special C# program that uses the Roslyn Syntax APIs to construct a syntax tree whose full text is equal to the input C# program. The goal of the Quoter tool is to demonstrate how to use the Roslyn Syntax APIs to generate syntax trees.
Example: for the input string "class C{}", the output will be:
C#
Edit|Remove
csharp 
Syntax.CompilationUnit()
.WithMembers(
    Syntax.List<MemberDeclarationSyntax>(
        Syntax.ClassDeclaration(
            @"C")))
.NormalizeWhitespace()
Syntax.CompilationUnit() 
.WithMembers( 
    Syntax.List<MemberDeclarationSyntax>( 
        Syntax.ClassDeclaration( 
            @"C"))) 
.NormalizeWhitespace()
 

Building the Sample

The C# project requires VS 2012 or later. It references Roslyn.Compilers.dll and Roslyn.Compilers.CSharp.dll. If you have the Roslyn September 2012 CTP installed (http://msdn.com/roslyn), then these binaries will be in the GAC.
Otherwise, you can use NuGet to add the Roslyn.Compilers.CSharp NuGet package, which will add these references automatically.
